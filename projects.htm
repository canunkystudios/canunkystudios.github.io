Hand Coded Game Engine with Multiplayer and Concurrency - C++, DirectX 11, Winsock for MSC.


YouTube

Was required to develop a game engine based around Entity - Component - System (ECS) architecture that was capable of running a game across a network.
A simple game, with a pyramid of balls - any number of layers from 3 - 50. A projectile could be fired from the camera to the center of the pyramid knocking any balls it hits off the pyramid.
The only simple lighting and texturing to be provided, and I implemented using DirectX 11.
I implemented an Octree for the physics which ran on its own thread, with double buffering and synchronization to allowed the process to not impede each other.
One game instance acted as a server, and all others connected as clients and Sockets (winsock) over TCP were used for messaging. Authentication and checksums were included on messages for ensuring message integrity. All sockets had their own Read and Write threads, placing and retrieving messages on a queuing system utilizing mutex locks to ensure thread and data safety. Each client can run at different network and graphics frame rates.
